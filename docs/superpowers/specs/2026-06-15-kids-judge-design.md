# ぎょぴ判定器 こども教材化 設計

作成 2026-06-15。口調=原始人(genshijin)。

## 目的

小学生向け 教育教材。狙い2つ:
1. **コンピュータがどう判断してるか 視覚的に見せる**
2. **「AIは間違えもするよ」を体感させる**(正直な自信メーター)

精度もある程度必要 → MobileNet モデルは維持。

## 対象

小学生。難語禁止。絵・直感・子供語。

## 全体思想(最終形・参考)

**2モード切替 教材**。同じ画像を2つの見方で比較:

- **モードA「どこ見た?」👀** = MobileNet + オクルージョン。高精度。分かるのは**位置だけ**
- **モードB「なぜ?」🎨** = 色/形の手作り特徴 + 小型分類器。**軸が言葉になる**。精度は落ちる

→ オチ「正確だけど理由不明(A) vs 理由分かるが間違える(B)」。AIの種類で見え方が違う、を体感。

最終形 追加要素(Phase2/3):
- **共通 判断カード**: 軸=場所/色/明るさ/鮮やか/かたち。使う軸=点灯、使わない=グレーアウト
- **脳みそネットワーク図**: 軸→脳細胞→こたえ。線が光って「組み合わせて考える」描写
- **%表示2種**: 「ぎょぴっぽさ%(一致度)」「効き%(寄与度)」

## レビュー決定事項(重要)

1. **Phase分割で実装**。一気に作らない。まず Phase 1 出荷
2. **「全項目どっち%」は諦め**。モードAは軸別%を原理的に出せない(ブラックボックス)。**軸別%はモードB専用**
3. **モードごとに見せ方が違う**のを受け入れる。モードA=場所マップ、モードB=軸グラフ。違い自体が教材オチ
4. **「効き%」は近似**と明示。relu非線形で厳密分解不能。重み×活性の近似。トーン下げる
5. 脳グラフの中間ノードは**小型(4〜8)**。Dense(64)は描くとぐちゃぐちゃ。Phase2で専用小型モデル要

## Phase 1(本spec の実装対象)

**スコープ = 単一モード(モードAのみ)を子供向け化。既存資産だけで動く教材を出荷。**

対象ファイル: `index.html` 改造。MobileNet・IndexedDB分類器・オクルージョンは**そのまま流用**。

### 1-0. 前提条件(重要)
- 判定には IndexedDB `gyopi-classifier`(学習済)が**必須**。無いと判定不可
- 初回は **train.html で学習が必須**。`index.html` の `loadModel` は未学習時 catch して「train.html で学習して」と出す(既存挙動)
- 教材配布時は「**先に学習ページで学習**」の導線が要る。未学習メッセージを子供語に直す

### 1-1. 子供向け 見た目・言葉
- タイトル・説明文 子供語
- でかいボタン「これ だれ?」
- 結果ドーン → 自信メーター → 場所マップ

### 1-2. 結果バッジ + 正直な自信メーター(役割分離)

**バッジとメーターは別の役割**。混ぜると矛盾る(旧 `showResult` は 0.50 単一境界で YES/NO → 迷い帯と衝突)。分ける:

**でかいバッジ(`result-badge`)= 結論。3値**。境界を 0.45 / 0.55 に変更(旧0.50単一を廃止):
- `prob >= 0.55` → 🐟「ぎょぴちゃんだ!」
- `0.45 <= prob < 0.55` → 🤔「どっちかな?」(=断定しない=迷い帯)
- `prob < 0.45` → 「ちがうみたい」

**自信メーター = どれくらい自信あるか。6段階文言**(既存 一致率バー 流用):
- `prob >= 0.90` → 「ぜったい ぎょぴちゃん!」
- `0.70 <= prob < 0.90` → 「たぶん ぎょぴちゃん」
- `0.55 <= prob < 0.70` → 「ぎょぴちゃん…かな?」
- `0.45 <= prob < 0.55` → 😵‍💫「うーん、まよう…」(=「間違えもする」体感帯)
- `0.30 <= prob < 0.45` → 「ちがう…かな?」
- `prob < 0.30` → 「ぎょぴちゃんじゃない」

迷い帯(0.45〜0.55)はバー色=黄「まよい中」。バッジ🤔 と一致。

### 1-3. 「どこ見た?」場所マップ(主役)
既存 `computeOcclusionMap` / `renderHeatmap` を子供向け改造:

- **青(逆方向)を消す**。赤〜黄だけ = 「ここを見て決めた場所」のみ。混乱減
- **①だけ大きく**「ここ!👀」。②③は**出さない**(決定。小学生は①のみで十分)
- 説明 子供語「コンピュータは ここを見て きめたよ」
- **注記1行**: 「※ほんとは いろも かたちも ぜんぶ見てるよ。でも コンピュータの あたまの中は のぞけないんだ」(モードAの誤解防止 + 正確さ担保)
- 計算中表示「コンピュータが かんがえてる…🤔」

### 1-4. 技術メモ(変えない)
- MobileNet埋込 + IndexedDB `gyopi-classifier` + オクルージョン = そのまま
- 追加は**文言マッピング・色・閾値**だけ。判定ロジック・精度に影響なし
- 既存 `judge()` `showResult()` `computeOcclusionMap()` `renderHeatmap()` を改造

### 1-5. モード切替トグル
**Phase 1 では置かない**(単一モードのみ)。YAGNI。Phase 2 で2モード化する時に追加。

### 1-6. Phase 1 完了条件(教材成立性チェック)
40枚モデル(ぎょぴ26/非14、非は連番=偏り)のまま動かす → **誤判定・"まよう"連発のリスク**。スコープ外で逃げ切れない。完了条件:

- 代表画像(ぎょぴ数枚・非ぎょぴ数枚)で **判定が概ね妥当か目視確認**
- 自信メーター文言が確率帯と一致
- 場所マップ表示・①ハイライト動作
- 未学習時メッセージが子供語で出る
- **判定が妥当でない(誤判定 or "まよう"連発)なら → 学習データ追加を Phase 1.5 に昇格**

## Phase 2(✅ 実装済み 2026-06-15)

- モードB = 色/形 手作り特徴 + **小型分類器を新規学習**(train.html 拡張)
  - 5特徴(warmth/brightness/saturation/edgeDensity/hueVariety)→ Dense(8,relu)→Dense(2,softmax)
  - IndexedDB `gyopi-feature-classifier` 保存。ぎょぴ群平均は localStorage `gyopi-avg-features`
- **モード切替トグル**(👀どこ見た? / 🎨なぜ?)。切替で同じ画像を再判定 → 見くらべ
- **判断カード**(モードB専用): 5軸 + 場所(グレー固定)。各軸に一致%/効き%
  - **一致%(ぎょぴっぽさ)** = `1 - |input - gyopiAvg|`
  - **効き%(きいた度)** = 各軸 ±eps の数値微分 → 正規化(近似)。効き低い軸は dim 表示
- 結果バッジ・自信メーターは Phase1 と共通

**実測(訓練データ自己評価)**: モードA acc 100% / モードB acc 65%。
→ 狙い通り「正確だが理由不明(A) vs 理由分かるが控えめ・間違える(B)」の対比が出る。
ぎょぴ判定例: A=100% / B=60%、判断カードは「いろの多さ きいた49%」が主・他はうすい。

### Phase 2 で見送った点(Phase 3 へ)
- 中間ノードの可視化(脳グラフ)は未実装 → Phase 3
- 中間8ノードは描画前提でないため小型化のみ(脳グラフ用の4〜8厳密化は Phase 3 で)

## Phase 3(✅ 実装済み 2026-06-15)

- **脳みそネットワーク図** canvas描画(`#brain-canvas`)。両モードで見た目が変わる=対比オチ
- **モードB(ほんものの脳)** = `featClassifier.getWeights()` で**実重み**を線の太さ/色に反映
  - 5入力(軸アイコン)→ 8中間 → 出力🐟。中間活性は手計算 relu。線色=重み正(coral)/負(sky)
  - いちばん効いた軸から **光パルス**が流れる(requestAnimationFrame)
- **モードA(のぞけない脳)** = 🖼️→ もやもや靄ノード40個 + 「?」→ 🐟。「中は見えない」を絵に
- モード切替で脳グラフも切替(古いアニメループは `_brainDraw` 比較で停止)

**実測**: 両モード block表示・描画OK。A=靄/B=実重み線、対比 成立。

## スコープ外

- **精度UP(学習データ追加)は別件**。Phase 1 は今の40枚モデルのまま動かす。教材的に「間違える」のはむしろ狙い通り

## 既知の課題

- オクルージョン粒度UP(5×5→7×7等)は推論回数=マス数で**重くなる**。子供の待ち時間に影響。Phase 1 は5×5維持。粒度UPは要検証
- 自信メーター閾値・一致%スケーリングは**魔法定数**。教材的近似と割り切る
- このプロジェクトは git 管理外。spec の commit はスキップ(必要なら `git init` 提案)
